# 🧠 Задание 5. Проектирование GraphQL API

---

## 1. 📦 Основные сущности из Swagger:

- `Client` — основная сущность клиента, которая содержит информацию о клиенте.

- `Document` — информация о документах клиента.

- `Relative` — информация о родственниках клиента.

`GraphQL идеально подходит под задачу: вместо "огромной" схемы REST с 500 полями, клиент сам выбирает, что ему нужно. Нет избыточных данных.`

- Необходимо внедрить нормализованный кэш Apollo Client
- Необходимо ограничить глубину запроса
- По **умолчанию** REST более безопасный
